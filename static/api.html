Starlight API

The API exports data from the game’s internal databases in a nice, webdev friendly JSON format.

Endpoints

/api/v1/(object type)/(spec)?...

Query the API for the objects matching spec.

Supported object types

  • skill_t: Card skill info.
  • card_t: Card info.
  • leader_skill_t: Centre skill info.
  • char_t: Character info.

Specs

A spec is a list of IDs separated by commas. IDs are not sequential, and are only valid for their object type. Typically, an ID is an integer, but there are some super-IDs that you can use:

  • Range ID: id1-id5 Matches all valid objects with an ID in the (inclusive) range.
  • All ID: all Matches every object of its type. If you pass it as part of a multi-ID spec, an error is returned.
  • Random ID: random Matches a random valid object. Duplication is not checked, so the spec random,random might return the same object twice.

Query Parameters

  • stubs=yes|no When stubs is present, linked objects (such as card_t.skill) are replaced with an {“ref”: url} object, where url is a valid API URL to retrieve the object that would normally be embedded. Returned URLs only contain the path and query portion, so you should add the scheme/domain yourself.

    {“ref”: “/api/v1/skill_t/123?stubs=yes”}
  • datetime=unix|iso8601 The format for timestamps returned by the API.
    • unix: Seconds since Jan 1, 1970 at midnight UTC as an integer.
    • iso8601: An ISO 8601 string: “1970–01–01T00:00:00.000Z”
